# 📸 이미지 업로드 기능 가이드

## 개요

증산도 경주 노서도장 게시판에서 **직접 이미지 파일을 업로드**할 수 있는 기능이 추가되었습니다!

## ✨ 주요 기능

### 1. 이미지 파일 업로드
- 💾 **파일 선택**: 컴퓨터나 휴대폰에서 이미지 파일 선택
- 👀 **미리보기**: 업로드 전 이미지 미리 확인
- 🗑️ **삭제**: 선택한 이미지 취소 가능

### 2. 이미지 URL 입력 (기존 방식)
- 🔗 외부 이미지 URL 직접 입력도 계속 사용 가능
- 📷 Unsplash, Imgur 등 외부 이미지 링크 지원

### 3. 썸네일 업로드
- 🖼️ 게시글 목록에 표시될 썸네일 이미지 업로드
- 📐 자동으로 적절한 크기로 표시

## 🎯 사용 방법

### 게시글 작성 시

1. **글쓰기 버튼 클릭**
2. **이미지 업로드 섹션에서**:
   - **방법 1**: "이미지 선택하기" 버튼 클릭 → 파일 선택
   - **방법 2**: "또는 이미지 URL 입력" 필드에 URL 입력
3. **선택한 이미지 미리보기 확인**
4. **썸네일도 같은 방식으로 업로드** (선택사항)
5. **게시하기 버튼 클릭**

### 이미지 삭제/변경

- 미리보기 이미지 우측 상단의 **❌ 버튼** 클릭
- 다른 이미지로 다시 선택 가능

## 📏 파일 제한사항

### 파일 크기
- **최대 크기**: 5MB
- **권장 크기**: 2MB 이하 (빠른 로딩을 위해)
- 크기 초과 시 경고 메시지 표시

### 파일 형식
- ✅ **지원 형식**: JPG, JPEG, PNG, GIF, WebP
- ❌ **미지원**: PDF, 동영상 파일 등

### 왜 5MB로 제한하나요?
- 📱 **모바일 사용자**: 데이터 절약
- ⚡ **빠른 로딩**: 페이지 로딩 속도 향상
- 💾 **저장 공간**: 데이터베이스 용량 관리

## 🔧 기술 정보

### Base64 인코딩 방식

이 웹사이트는 **Base64 인코딩** 방식을 사용합니다:

#### 장점
- ✅ 별도의 파일 서버 불필요
- ✅ 즉시 업로드 및 표시
- ✅ 추가 비용 없음
- ✅ 설정 불필요

#### 단점
- ⚠️ 파일 크기 제한 (5MB)
- ⚠️ 대용량 이미지 업로드 시 시간 소요
- ⚠️ 데이터베이스 용량 증가

### 이미지 최적화 팁

업로드 전 이미지를 최적화하면 더 빠른 로딩이 가능합니다:

#### 1. 온라인 도구 사용
- [TinyPNG](https://tinypng.com/) - PNG/JPG 압축
- [Squoosh](https://squoosh.app/) - Google의 이미지 최적화 도구
- [Compressor.io](https://compressor.io/) - 다양한 형식 지원

#### 2. 권장 이미지 크기
- **게시글 이미지**: 1200px × 800px 이하
- **썸네일**: 600px × 400px 이하
- **해상도**: 72 DPI (웹용)

#### 3. 파일 형식 선택
- **사진**: JPG (파일 크기 작음)
- **그래픽/로고**: PNG (투명 배경 지원)
- **애니메이션**: GIF

## 📱 휴대폰에서 업로드

### 사진 찍어서 바로 업로드

1. 글쓰기 화면에서 "이미지 선택하기" 클릭
2. **카메라** 선택 (또는 **갤러리**에서 선택)
3. 사진 촬영 또는 기존 사진 선택
4. 자동으로 미리보기 표시
5. 게시하기

### 휴대폰 사진 파일 크기 줄이기

#### 안드로이드
1. 갤러리 앱에서 사진 선택
2. 편집 → 크기 조정
3. 해상도 낮추기 (예: 1920×1080)

#### iOS (iPhone)
1. 사진 앱에서 선택
2. 공유 → 옵션
3. "실제 크기" 대신 "중간" 또는 "작게" 선택

## 🚨 문제 해결

### 1. "이미지 파일 크기는 5MB 이하여야 합니다" 오류

**원인**: 업로드하려는 이미지가 5MB보다 큼

**해결방법**:
1. 위에 소개된 온라인 도구로 이미지 압축
2. 이미지 편집 프로그램에서 해상도 낮추기
3. 다른 이미지 선택

### 2. "이미지 파일을 읽는 중 오류가 발생했습니다"

**원인**: 파일이 손상되었거나 지원되지 않는 형식

**해결방법**:
1. 다른 이미지 파일 선택
2. 이미지를 JPG나 PNG로 다시 저장
3. 파일 이름에 특수문자가 있다면 제거

### 3. 업로드가 너무 느림

**원인**: 파일 크기가 크거나 인터넷 속도가 느림

**해결방법**:
1. 이미지 크기를 2MB 이하로 압축
2. Wi-Fi 환경에서 업로드
3. 이미지 해상도를 낮추기

### 4. 미리보기에서 이미지가 안 보임

**원인**: 
- URL이 잘못됨
- 이미지 서버에 접근 불가
- HTTPS가 아닌 HTTP URL

**해결방법**:
1. URL이 정확한지 확인
2. 브라우저에서 URL을 직접 열어 확인
3. HTTPS URL 사용
4. 파일 업로드 방식으로 전환

### 5. 게시 후 이미지가 안 보임

**원인**: Base64 데이터가 너무 크거나 손상됨

**해결방법**:
1. 이미지 크기를 줄여서 다시 업로드
2. 외부 이미지 호스팅 서비스 사용 (Imgur 등)
3. 다른 형식(JPG → PNG)으로 변환 시도

## 🌟 권장 사항

### 일반 게시글
- 📸 **이미지 크기**: 1-2MB
- 📐 **해상도**: 1200×800px 이하
- 🎨 **형식**: JPG (사진), PNG (그래픽)

### 행사사진
- 📸 **이미지 크기**: 2-3MB
- 📐 **해상도**: 1920×1080px
- 🎨 **형식**: JPG
- 💡 **팁**: 여러 장 올릴 경우 각각 2MB 이하로

### 썸네일
- 📸 **이미지 크기**: 500KB 이하
- 📐 **해상도**: 600×400px
- 🎨 **형식**: JPG
- 💡 **팁**: 게시글 대표 이미지로 선택

## 🔮 향후 개선 계획

### 현재 Base64 방식의 한계

대용량 이미지나 많은 이미지를 업로드하면 데이터베이스 용량이 증가하고 페이지 로딩이 느려질 수 있습니다.

### 향후 업그레이드 옵션

1. **클라우드 스토리지 연동**
   - Cloudinary, AWS S3 등
   - 무제한 용량
   - 자동 이미지 최적화
   - CDN으로 빠른 로딩

2. **이미지 자동 최적화**
   - 업로드 시 자동 압축
   - 다양한 크기로 자동 변환
   - WebP 형식 자동 변환

3. **드래그 앤 드롭**
   - 이미지 파일 드래그하여 업로드
   - 여러 이미지 한번에 업로드

## 💡 팁과 요령

### 빠른 업로드를 위한 팁

1. **이미지 사전 준비**
   - 업로드 전에 미리 압축
   - 적절한 해상도로 리사이즈
   - 불필요한 부분 크롭

2. **Wi-Fi 사용**
   - 모바일 데이터 대신 Wi-Fi 사용
   - 안정적인 네트워크 환경

3. **한번에 하나씩**
   - 여러 이미지는 순차적으로 업로드
   - 업로드 완료 후 다음 이미지

### 좋은 썸네일 만들기

1. **명확한 주제**
   - 게시글 내용을 대표하는 이미지
   - 텍스트보다는 이미지 중심

2. **적절한 구도**
   - 중요한 요소가 중앙에
   - 과도한 여백 제거

3. **밝고 선명하게**
   - 너무 어두운 이미지 피하기
   - 흐릿한 이미지 피하기
